# Локальный AI-агент (Ollama + инструменты)

Минимальный агент на Python: цикл «мысль → действие → наблюдение», вызов зарегистрированных инструментов и финальный ответ в JSON. Модель и базовый URL Ollama задаются через переменные окружения; **имя модели (`OLLAMA_MODEL`) — единственный источник истины** для запуска.

## Требования к окружению

- Python **3.12+** (проверено также на 3.13).
- Установленный [Ollama](https://ollama.com/) и локально скачанная модель (см. ниже).

## Установка зависимостей

Рекомендуется виртуальное окружение:

```bash
cd ai-agent-hw
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Скопируйте пример конфигурации:

```bash
cp .env.example .env
```

Отредактируйте `.env` при необходимости (особенно `OLLAMA_MODEL`).

Системный промпт выбирается переменной **`SYSTEM_PROMPT_PRESET`**:

- `default` — короткая инструкция из исходной спеки (по умолчанию);
- `autonomous` — расширенный текст с циклом агента, правилами и плейсхолдером списка инструментов (см. `app/services/system_prompt_templates.py`).

Пример:

```text
SYSTEM_PROMPT_PRESET=autonomous
```

## Ollama и модель

1. Установите Ollama по инструкции с официального сайта.
2. Скачайте выбранную модель (в `.env.example` приведён пример `qwen3.5:4b`):

   ```bash
   ollama pull qwen3.5:4b
   ```

3. Убедитесь, что Ollama слушает API (по умолчанию `http://localhost:11434`). Сервис обычно поднимается при первом запросе или при запуске приложения Ollama.

## Запуск агента

Из корня проекта, с активированным venv.

**Режим «вопрос — ответ» (интерактивный REPL):** запустите без аргументов — программа будет запрашивать задачи по одной, пока вы не введёте **пустую строку** (или не закроете ввод, например Ctrl+D):

```bash
python -m app.main
```

После каждого ввода агент выполнит цикл и выведет шаги и ответ; затем снова появится `Enter task:`.

**Один запуск с аргументом** (без диалога):

```bash
python -m app.main "Посчитай (123 + 456) * 2"
```

Код выхода: `0` при успешном `final_answer` в одноразовом режиме; в интерактивном режиме после выхода по пустой строке — `0`, если не было неуспешных прогонов, иначе `2` (если хотя бы один прогон завершился с ошибкой).

## Примеры задач

- Арифметика: `Посчитай (123 + 456) * 2` — ожидается вызов `calculator` и финальный ответ.
- Файл: `Прочитай файл workspace/test.txt и скажи сколько там строк` — `readFile` внутри каталога `WORKSPACE_PATH` (по умолчанию `./workspace`).
- HTTP: `Сделай запрос к https://example.com и верни первые 200 символов ответа` — `httpGet` с ограничением длины ответа.

## Ограничения текущей версии

- Нет повторного запроса к модели при невалидном JSON (MVP: понятная ошибка).
- HTTP-инструмент: только GET, без фильтрации URL (SSRF).
- Чтение файлов только под каталогом `WORKSPACE_PATH`; пути с `..` и выход за пределы каталога отклоняются.
- Агент зависит от того, что модель строго возвращает указанный JSON (см. промпт в `app/services/prompt_builder.py`).

## Тесты

```bash
source .venv/bin/activate
python -m unittest discover -s tests -v
```

Тесты **не требуют** запущенного Ollama: клиент LLM подставляется заглушкой в тестах раннера; HTTP и файлы проверяются изолированно.

## Структура (кратко)

- `app/domain` — сущности и протоколы.
- `app/services` — раннер, сборка промпта, разбор ответа, реестр инструментов.
- `app/infrastructure` — Ollama HTTP-клиент и инструменты.
- `workspace/` — рабочая область для `readFile`.
